# スライド用draw.ioレイアウト

## 目的

1280x720のMarpページへ、タイトル・header・footerを含めて安全に収まる横長の構造図を作る。

## 構図

- 基本方向は左から右。階層図だけ上から下を使う
- 図の外接矩形は横:縦がおおむね`1.6:1`以上になるよう組む。縦長ならページ分割か方向変更を検討する
- 単独図は横幅960〜1000px相当、結論文や注記と同居する図は900px以下を目安にする
- 外周へ10〜24px相当の余白を残し、ノードや矢印を端へ接触させない
- 1ページの主役は1図。本文リストと大きな図を左右へ詰め込まない

## 情報量

- 1図の主要ノードは原則5〜12個。12個を超えたら概要図と詳細図へ分ける
- 1ノードは見出し1行、補足が必要でも2行まで
- ノード内で文章を説明しない。詳細は本文ページへ分離する
- 線が交差するなら、配置変更、境界によるグルーピング、図の分割の順で直す

## 文字と色

- 通常ラベルは16px相当以上、主要ノードは18〜20px相当を起点にする
- 色数は意味のある3〜5色まで。ZENSHINテーマへ無理に寄せず、白背景上の可読性を優先する
- 良い／悪い、成功／失敗など意味がある場合だけ緑／赤を使う
- 白抜き文字は濃色面だけに限定し、薄色面では濃紺または黒を使う

## 最終確認

SVG単体ではなく、Marpから生成した該当ページのPNG/PDFを見る。

1. タイトルと図の間に呼吸がある
2. 図の上下左右が切れていない
3. 最小ラベルが100%表示で読める
4. 矢印の向きと接続先が曖昧でない
5. 図の結論がタイトルと重複せず補完関係になっている
